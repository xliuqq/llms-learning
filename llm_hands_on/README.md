## 4. 文本分类
> [METB 排行榜](https://huggingface.co/spaces/mteb/leaderboard)：选择生成嵌入向量的模型时。

> 分类模型的指标评价（混淆矩阵）：精确率，召回率，F1 score.

利用预训练的语言模型来完成文本分类任务，同时研究表示模型和生成模型。

表示模型：特定任务模型和嵌入模型（BERT系列）
- 特定任务模型是一种表示模型，它针对特定任务（如情感分析）进行训练，直接出来类别信息；
- 嵌入模型可以生成通用嵌入向量，可以用于各种任务；
  - 嵌入模型对训练数据输出嵌入向量，输入适配网络（如LR进行二分类），仅训练 LR 网络；
  - 零样本分类：对分类的类别给文本描述，再生成嵌入向量，跟预测的文本的嵌入向量计算余弦相似值，取最大值的类别。
   
生成模型：编码器-解码器模型（FLAN-T5）和仅解码器模型（GPT-3.5）
- T5 模型：对词元集合（token span）整体进行掩码，再将每个任务转换为序列到序列任务并**同时**进行训练。
- ChatGPT 模型：使用指令数据（输入提示词和期望输出）创造模型第一个版本，再用该模型生成生成多个输出并进行排序
  得到偏好数据，再进行训练得到最终的模型；通过偏好数据，生成模型学会生成符合人类偏好的文本。

## 5. 文本聚类

文本聚类不受监督约束，可以实现创造性解决方案和多样化应用，如寻找离群点、加速标注和发现标注错误的数据。

主题建模：再大量文本数据集合中发现（抽象的）主题时发挥作用，使用关键词或者关键短语来描述主题。

主流的文本聚类的步骤：
1. 使用**嵌入模型**将输入文档转换为嵌入向量；
   - 选择为**[语义相似度任务优化的嵌入模型](https://hf-mirror.com/thenlper/gte-small)**对聚类任务特别重要。
2. 使用**降维模型**（PCA/UMAP）将嵌入向量降至更低维度空间；
   - UMAP（统一流形逼近和投影）在处理非线性关系和结构方面比 PCA 表现更好
3. 使用**聚类模型**将降维后的嵌入向量进行聚类。
   - kmeans：基于质心的算法，需要生成一组预设的簇；但基于密度的算法可以自由计算簇的数量，不强制所有数据点属于某个簇；
   - HDBSCAN（具有噪声的分层密度空间聚类）：基于 DBScan 的层次化变体，还可以检测数据中的离群点（不属于任何簇）

主题建模：在文本数据集合中寻找主题或者潜在语义的思路，寻找一组最能代表、捕捉主题含义的关键词或短语。
- 经典方法 LDA（潜在狄利克雷分配），假设每个主题都由语料库词表中的概率分布来表示，通常使用词袋技术提取文本数据特征，没有考虑词和短语的上下文及含义。
- BERTopic：高度模块化的文本聚类和主题建模框架。

[BERTopic](https://maartengr.github.io/BERTopic/getting_started/best_practices/best_practices.html)：模块化主题建模框架，利用语义相似的文本聚类来提取各种类型的主题表示。流程分为两个部分：
- 聚类：对文档进行嵌入、降维、聚类，创建语义相似的文档组；
- 主题表示：利用经典的词袋方法，对此表中的词再语料库中的分布建模。

BERTopic 中的主题表示，采用基于类的词频-逆文档频率（c-TF-IDF=c-TF * IDF）进行词向量表示，提升对单个簇更有意义的词的权重，降低在所有簇中都常用的词的权重。

c-TF-IDF 跟传统的 TF-IDF 不同，关注“类内活跃度”和“跨类稀有性”（IDF，关注的**跟其他词的比较**，而不是在簇中出现的概率）
- c-TF:词在整个簇中而不是单个文档中的词频；
- IDF：$log(\frac{A}{cf_x} + 1)$, 所有簇中所有词的平均频次（A）除以当前词的所有总频次($cf_x$}， 加1，再取对数；

表示模型：BERTopic 默认通过词袋模型来表示主题，没有考虑语义结构，因此需要对词的初始（c-TF-IDF）分布进行重新排序，改进最终的表示。
- KeyBERTInspired: 受关键词提取包KeyBERT启发，更新后的主题关键词更容易阅读，但是领域专有词 mnt（神经网络翻译）被移除，且类似的词出现多次（冗余），如 translate/translation/traslations:
  - 提取每个主题中最具代表性的 N 个文档，随机选取 M 篇文档，通过计算**文档的 c-TF-IDF 向量**与其对应的主题的 c-TF-IDF 向量的相似度（按照词表为维度，为每个词计算）；
  - N 个文档的嵌入向量做平均值计算，得到该**主题语义特征向量表示**，并跟主题的 Top N 关键词的语义嵌入（embedding）计算余弦相似度，并排序；
- 最大边际相关性（Maximal Marginal Relevance，MMR）：使得主题更加多样化，寻找一组相互之间具有差异性、但仍与所比较的文档相关的关键词；
  - $\text{MMR}(S, Q, \lambda) = \arg\max_{d \in S \setminus S_{\text{selected}}} \left[ (1 - \lambda) \cdot \text{Sim}(d, Q) - \lambda \cdot \max_{d' \in S_{\text{selected}}} \text{Sim}(d, d') \right]$
  - S 表示待选择的候选词（嵌入向量）集合，Q 表示主题的嵌入向量，d 表示待评估的词，Sim 表示相似度计算函数（如余弦）；
  - 嵌入一组候选关键字，并迭代计算下一个最佳关键字，需要设置多样性参数用于指示关键词需要多大的差异性。

遵循重排序的过程，在 BERTopic 中使用生成模型为主题生成标签，要求模型基于先前生成的关键词和一小组典型文档生成简短的标签。
- 通过 'DOCUMENTS' 标签插入最能代表该主题的文档，通常4个，基于其 c-TF-IDF 值与主题的余弦相似度最高选择；
- 将构成主题的关键字也传给提示词，并使用 'KEYWORDS' 标签引用，可以由 c-TF-IDF 或之前讨论的任何表示方法生成；

## 9. 多模态 LLM

多模态模型：能够处理文本和图像（每种数据类型称为一种模态）的模型。

ViT(Vision Transformer) ：将图形切割为“词”的策略，使用原始 Transformer 的编码器架构特性。
- 将原始图像切割为规则排列的图像块（如16*16），沿着图像的水平方向和垂直方向进行网格化切割，将图像块展平输入可被视为视觉领域的“词元”，
  但由于图像的多样性，图像块重复率极低，无法直接赋予图像块固定的ID；
- 直接对图像块实施**线性嵌入操作，将其转化为数值化的嵌入向量**，作为 Transformer 模型的标准化输入；
